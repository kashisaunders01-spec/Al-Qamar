Setup guide

Everything lives in this one file plus your Supabase project. Steps 1–2 make the log work; step 3 turns on the email reminders.

1Create the tables (once)

In Supabase: SQL Editor → New query, paste this, press Run. Safe to re-run — it also upgrades any older version of the tables (adding the supplier column, engine-hours support, and the fuel log).

-- AL QAMAR — one-time setup (safe to re-run; migrates older versions)

create table if not exists public.service_items (
  id                  uuid primary key default gen_random_uuid(),
  name                text not null,
  category            text not null default 'General',
  interval_type       text not null default 'days',   -- 'days' | 'hours'
  last_serviced       date not null default current_date,
  life_days           integer check (life_days > 0),
  engine              text,                           -- 'port' | 'starboard'
  life_hours          numeric check (life_hours > 0),
  last_serviced_hours numeric,
  supplier_url        text not null default '',
  notes               text not null default '',
  reminder_sent       boolean not null default false,
  created_at          timestamptz not null default now(),
  updated_at          timestamptz not null default now()
);

-- upgrade path from earlier versions of this script
alter table public.service_items add column if not exists supplier_url text not null default '';
alter table public.service_items add column if not exists interval_type text not null default 'days';
alter table public.service_items add column if not exists engine text;
alter table public.service_items add column if not exists life_hours numeric;
alter table public.service_items add column if not exists last_serviced_hours numeric;
alter table public.service_items alter column life_days drop not null;

-- engine hour-meter readings (port & starboard)
create table if not exists public.engine_hours_log (
  id        uuid primary key default gen_random_uuid(),
  engine    text not null check (engine in ('port','starboard')),
  hours     numeric not null check (hours >= 0),
  logged_at timestamptz not null default now()
);

-- diesel fills (shared tank — consumption is combined)
create table if not exists public.fuel_log (
  id         uuid primary key default gen_random_uuid(),
  fill_date  date not null default current_date,
  liters     numeric not null check (liters > 0),
  port_hours numeric,
  stb_hours  numeric,
  created_at timestamptz not null default now()
);

create table if not exists public.app_settings (
  id             integer primary key default 1 check (id = 1),
  reminder_email text
);
insert into public.app_settings (id) values (1)
  on conflict (id) do nothing;

create or replace function public.touch_updated_at()
returns trigger language plpgsql as $$
begin new.updated_at = now(); return new; end $$;

drop trigger if exists service_items_touch on public.service_items;
create trigger service_items_touch
  before update on public.service_items
  for each row execute function public.touch_updated_at();

-- single-user access: the anon key gets full read/write
alter table public.service_items    enable row level security;
alter table public.engine_hours_log enable row level security;
alter table public.fuel_log         enable row level security;
alter table public.app_settings     enable row level security;

drop policy if exists "anon full access items"    on public.service_items;
drop policy if exists "anon full access hours"    on public.engine_hours_log;
drop policy if exists "anon full access fuel"     on public.fuel_log;
drop policy if exists "anon full access settings" on public.app_settings;

create policy "anon full access items" on public.service_items
  for all to anon using (true) with check (true);
create policy "anon full access hours" on public.engine_hours_log
  for all to anon using (true) with check (true);
create policy "anon full access fuel" on public.fuel_log
  for all to anon using (true) with check (true);
create policy "anon full access settings" on public.app_settings
  for all to anon using (true) with check (true);

The anon key is visible in the page source, and anyone holding it can edit your log. Fine for a personal boat log — just don't post the link publicly.

3Email reminders (optional, ~10 min)

A web page can't wake itself up to email you, so Supabase does it: a small function runs daily and emails the address saved in the log's Settings when anything is within 7 days (calendar items) or 10 engine-hours (hour items — e.g. at 190 h of a 200 h interval) of due. Marking an item serviced re-arms its reminder — no double nagging.

Hour-based reminders can only see the meters you log. The engines don't phone home — so log your hours after each passage or at every fuel fill (the fuel form does it for you), and the daily email job stays accurate.
daily 08:00 UTC ──▶ pg_cron ──▶ Edge Function "send-reminders" ──▶ Resend ──▶ your inbox

3a. Sign up free at resend.com → API Keys → create one (re_...). Until you verify a domain, Resend only delivers to the email you signed up with — which is exactly right for a personal reminder.

3b. Supabase → Edge Functions → Deploy a new function (use the in-browser editor). Name it exactly send-reminders, paste this, deploy. Already deployed an older version? Paste over it and redeploy — this one understands engine hours.

import { createClient } from "npm:@supabase/supabase-js@2";
const DAY = 86400000;
const DAYS_BEFORE = 7;    // calendar items: remind 7 days out
const HOURS_BEFORE = 10;  // hour items: remind 10 engine-hours out (190 of 200)

Deno.serve(async () => {
  const supabase = createClient(
    Deno.env.get("SUPABASE_URL")!,
    Deno.env.get("SUPABASE_SERVICE_ROLE_KEY")!,
  );

  // where to send
  const { data: settings } = await supabase
    .from("app_settings").select("reminder_email").eq("id", 1).single();
  const to = settings?.reminder_email;
  if (!to) return new Response("No reminder email set in app settings.");

  // latest hour-meter reading per engine
  const { data: hourRows } = await supabase
    .from("engine_hours_log").select("engine,hours,logged_at")
    .order("logged_at", { ascending: false }).limit(200);
  const meters: Record<string, number> = {};
  for (const r of hourRows ?? []) if (!(r.engine in meters)) meters[r.engine] = Number(r.hours);

  // items not yet notified
  const { data: items, error } = await supabase
    .from("service_items").select("*").eq("reminder_sent", false);
  if (error) return new Response(error.message, { status: 500 });

  const today = new Date(); today.setUTCHours(0, 0, 0, 0);
  const due: any[] = [];
  for (const i of items ?? []) {
    if (i.interval_type === "hours") {
      const cur = meters[i.engine];
      if (cur == null || i.last_serviced_hours == null || i.life_hours == null) continue;
      const left = Number(i.last_serviced_hours) + Number(i.life_hours) - cur;
      if (left <= HOURS_BEFORE) due.push({ ...i, kind: "hours", left,
        dueLabel: `at ${(Number(i.last_serviced_hours) + Number(i.life_hours)).toLocaleString()} h (${i.engine})`,
        countdown: left < 0 ? `${Math.abs(left).toFixed(1)} h overdue` : `${left.toFixed(1)} h left` });
    } else {
      const dueDate = new Date(new Date(i.last_serviced + "T00:00:00Z").getTime() + i.life_days * DAY);
      const left = Math.round((dueDate.getTime() - today.getTime()) / DAY);
      if (left <= DAYS_BEFORE) due.push({ ...i, kind: "days", left,
        dueLabel: dueDate.toISOString().slice(0, 10),
        countdown: left < 0 ? `${Math.abs(left)} days overdue` : `${left} days left` });
    }
  }
  due.sort((a, b) => a.left - b.left);
  if (due.length === 0) return new Response("Nothing due within the reminder window.");

  const link = (u: string) => /^https?:/i.test(u) ? u : "https://" + u;
  const rows = due.map((i) => `
    <tr>
      <td style="padding:8px 12px;border-bottom:1px solid #e5e5e5;">
        <b>${i.name}</b><br><span style="color:#777;font-size:12px;">${i.category}</span>
        ${i.supplier_url ? `<br><a href="${link(i.supplier_url)}" style="font-size:12px;color:#8A6A2E;">Order part →</a>` : ""}
      </td>
      <td style="padding:8px 12px;border-bottom:1px solid #e5e5e5;">${i.dueLabel}</td>
      <td style="padding:8px 12px;border-bottom:1px solid #e5e5e5;color:${i.left < 0 ? "#C24A33" : "#B4831C"};">
        ${i.countdown}
      </td>
    </tr>`).join("");

  const html = `
    <div style="font-family:Georgia,serif;max-width:560px;margin:auto;">
      <h2 style="color:#0F2E3A;">⚓ Al Qamar — service due</h2>
      <p>${due.length} item(s) need attention:</p>
      <table style="border-collapse:collapse;width:100%;font-family:Arial,sans-serif;font-size:14px;">${rows}</table>
      <p style="color:#777;font-size:12px;margin-top:16px;">Mark items as serviced in the log to reset their countdown.</p>
    </div>`;

  const res = await fetch("https://api.resend.com/emails", {
    method: "POST",
    headers: {
      Authorization: `Bearer ${Deno.env.get("RESEND_API_KEY")}`,
      "Content-Type": "application/json",
    },
    body: JSON.stringify({
      from: "Al Qamar <onboarding@resend.dev>",
      to: [to],
      subject: `⚓ Al Qamar — ${due.length} item(s) due for service`,
      html,
    }),
  });
  if (!res.ok) return new Response(await res.text(), { status: 500 });

  // don't remind about the same service twice
  await supabase.from("service_items")
    .update({ reminder_sent: true })
    .in("id", due.map((i) => i.id));

  return new Response(`Reminder sent to ${to} for ${due.length} item(s).`);
});

3c. Edge Functions → Secrets → add RESEND_API_KEY = the key from 3a.

3d. Schedule it daily: back in the SQL Editor, run this once (swap in your anon key). If you already scheduled it before, you don't need to run it again.

create extension if not exists pg_cron;
create extension if not exists pg_net;

select cron.schedule(
  'bosun-daily-reminders',
  '0 8 * * *',   -- 08:00 UTC daily = 17:00 Tokyo / 18:00 Sydney
  $$
  select net.http_post(
    url     := 'https://lrincvupqdlbsojdbuoa.supabase.co/functions/v1/send-reminders',
    headers := jsonb_build_object(
      'Authorization', 'Bearer YOUR_ANON_KEY',
      'Content-Type', 'application/json'
    ),
    body    := '{}'::jsonb
  );
  $$
);

3e. Test it now: Edge Functions → send-reminders → Invoke. Anything inside the reminder window lands in your inbox right away.

If no email arrives: check the function's Logs tab, confirm the reminder address is saved in the log's Settings, and that it matches the email you signed up to Resend with. For hour items, also check that engine hours have been logged recently.
